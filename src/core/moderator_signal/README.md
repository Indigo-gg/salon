# 主持人信号感知与注入系统

## 1. 系统概述

本模块是 Salon 系统中主持人 Agent 的"感知神经系统"。它将对话的物理状态建模为可观测的信号量，通过状态观测器映射为控制指令，最终注入主持人 LLM 的 prompt 中，辅助主持人做出更精准的控场决策。

**核心设计思想：** 主持人 LLM 本身是一个极强的"传感器"，它能感知对话的氛围、方向、深度。但它有两个弱点：
1. **数不准** — 连续几轮没有故事了？话语权基尼系数是多少？LLM 不擅长精确计数
2. **看不出渐变** — 可读性是在缓慢上升还是在下降？LLM 容易忽略微小的趋势变化

本系统的作用是**补盲**，而不是替代 LLM 的感知。

### 变更记录：调度器三层防线（2026-06-04）

#### 解决的问题

67 轮实际哲学对话（"推动人类发展的动力是什么？"）暴露了三个调度缺陷：

1. **CONVERGENCE 反复失败** — 讨论在 CONVERGENCE↔DEEPENING 之间震荡 5 次，主持人没有递增加强收敛压力，每次都被参与者拉回深入讨论
2. **原始问题追踪太晚** — 主持人在第 39 轮（67 轮的 58% 处）才第一次提醒"原始问题尚未收束"，中间被局部争论牵着走了很久
3. **发言权缺少平衡** — Albert 发言 118 条，Carl/Dawen 仅 96 条（差距 23%），系统没有主动纠正的机制

#### 设计原则

> **第 3 层只拦截"LLM 明显犯错"的情况，不拦截"LLM 做了你不喜欢的判断"。后者应该通过第 1 层改善信息质量来引导。**

- **公平但不平等** — 不强制 round-robin，而是给主持人更好的信息让它自行平衡
- **高重复 = 该转移，不是该结束** — 论点重复意味着当前子话题挖到底了，正确响应是转向新子命题，而非强制 CLOSING
- **CLOSING 只由时间驱动** — 用 `sqrt(max_rounds)` 曲线计算尾声窗口，上界 5 轮

## 3. 四个输入维度

### 3.1 方向（Direction）— 话题聚焦度

**测量的问题：** 讨论是否在主题的引力场内？

**测量方法：**
- 提取初始主题的关键词集合（分词后过滤停用词）
- 提取当前轮发言的关键词集合
- 计算主题关键词覆盖率（Jaccard 变体）
- coverage 高且新词少 = 紧扣主题

**输出值：** `topic_keyword_overlap` (0-1)，1 = 完全紧扣主题

**为什么不用 Embedding：** Embedding 语义相似度更精确，但需要加载 ~500MB 模型。关键词重叠作为第一层实现，零额外依赖，对"话题是否跑偏"这个任务精度够用。

### 3.2 高度（Height）— 抽象度

**测量的问题：** 讨论悬浮在多高的海拔上？

**测量方法（三个子指标加权）：**

| 子指标 | 权重 | 计算方式 | 物理意义 |
|--------|------|----------|----------|
| IDF 均值 | 50% | 词在通用语料库中的逆文档频率 | 高 IDF = 低频/专业词汇 |
| 抽象词占比 | 30% | IDF > 4.0 的词占比 | 直接衡量专业术语密度 |
| 锚点密度 | 20% | 数字/日期/引号/具体标记词 | 具体锚点多 → 高度低 |

**IDF 表来源：** 模块内置 ~300 个词的 IDF 值（覆盖常见哲学/学术词汇）。未收录的词使用基于词长的启发式估计（1 字=1.0, 2 字=2.0, 4 字=4.5, 5+字=5.5）。

**通用性：** IDF 表不绑定特定领域。任何领域的专业术语都是低频词，都会获得高 IDF 值。

### 3.3 速度（Speed）— 代谢率

**测量的问题：** 对话状态变化的速率有多快？

**测量方法（三个子指标加权）：**

| 子指标 | 权重 | 计算方式 | 物理意义 |
|--------|------|----------|----------|
| 概念代谢率 | 40% | 1 - (被引用的活跃概念 / 总活跃概念) | 高代谢 = 概念快速更替 |
| KL 散度 | 40% | 前后轮词频分布的 KL(P\|\|Q) | 高 KL = 信息增量大 |
| 句长 | 20% | 平均句长归一化 | 长句 = 信息密度高 |

**KL 散度的直觉：** 如果上一轮讨论"自由意志"，这一轮突然讨论"量子力学"，词频分布差异大 → KL 散度高 → 速度快。

### 3.4 阵型（Formation）— 群体结构

**测量的问题：** 参与者之间的互动结构是什么样的？

**测量方法（三个子指标加权）：**

| 子指标 | 权重 | 计算方式 | 物理意义 |
|--------|------|----------|----------|
| 引用密度 | 40% | 正则匹配引用模式的发言占比 | 高引用 = 真正在对话 |
| 立场分化 | 30% | 反驳+赞同标记词密度 | 高分化 = 有交锋 |
| Gini 系数 | 30% | 发言字数的基尼系数 | 高 Gini = 权力集中 |

**引用检测模式（正则）：**
- "你说的"、"正如X提到"、"正如X所说"、"回应X"、"针对X"
- "X的观点"、"X说的"、"X提到的"

**立场检测模式（标记词）：**
- 反驳："但是"、"然而"、"恰恰相反"、"我不认为"、"问题是"
- 赞同："确实"、"同意"、"没错"、"有道理"、"补充"

**Gini 系数：** 经济学中的不平等衡量指标。0 = 所有人发言字数完全相等，1 = 一个人说了所有话。

## 4. 四个变化率（Δ）

使用**指数移动平均（EMA）**计算：

```
EMA_α = 1 - 2^(-1/half_life)

ema_short(t) = α_short × current + (1 - α_short) × ema_short(t-1)   # 半衰期 3 轮
emA_long(t)  = α_long  × current + (1 - α_long)  × ema_long(t-1)    # 半衰期 10 轮

Δ = ema_short - ema_long
```

**直觉：**
- `Δ > 0`：短期值高于长期基线 → 正在上升
- `Δ < 0`：短期值低于长期基线 → 正在下降
- `|Δ|` 大：变化剧烈；`|Δ|` 小：趋于稳定

**为什么用 EMA 而不是简单差值：** EMA 对噪声有天然的平滑作用，不会因为某一轮的异常值产生剧烈波动。

## 5. 五个控制信号

### 5.1 可读性警报（Readability Alert）

```
readability_alert = height × (1 + max(0, Δheight))
```

- 高度越高 → 警报越强
- 高度在上升 → 警报进一步加强
- 高度在下降 → 警报自然缓解

**阈值：** > 0.7 触发主持人干预

### 5.2 深度潮汐（Depth Tide）

```
if height > 0.6 and speed > 0.4:
    连续 dive 计数 +1
elif height < 0.35 and speed < 0.35:
    连续 surface 计数 +1
else:
    两个计数归零

if 连续 dive >= 3: 信号 = "dive"（需要浮出）
if 连续 surface >= 3: 信号 = "surface"（需要深潜）
else: 信号 = "balanced"
```

**直觉：** 讨论在"高空高速"状态下持续多轮 → 需要安排浮出（故事/比喻）。讨论在"低空低速"状态下持续多轮 → 需要安排深潜（理论追问）。

### 5.3 话题聚焦警报（Topic Focus Alert）

```
topic_focus_alert = (1 - direction) × (1 + max(0, -Δdirection))
```

- 跑偏 → 警报激活
- 正在加速跑偏 → 警报加强
- 正在自然回归 → 警报缓解

**阈值：** > 0.6 触发主持人干预

### 5.4 对话张力（Dialogue Tension）

| 级别 | 条件 | 含义 |
|------|------|------|
| monologue | formation < 0.3 且方向 > 0.5 | 独白垄断 |
| parallel | formation < 0.3 且方向 ≤ 0.5 | 各说各话 |
| moderate | 其他情况 | 正常 |
| debate | formation > 0.7 且情绪温度 ≤ 0.7 | 正在交锋 |
| heated | formation > 0.7 且情绪温度 > 0.7 | 激烈对抗 |

**优先使用 LLM 反馈：** 如果主持人 LLM 报告了明确的张力感知（low/high/conflict），以 LLM 为准。

### 5.5 认知能量（Cognitive Energy）

| 级别 | 条件 | 含义 |
|------|------|------|
| exhausted | Δspeed < -0.15 且 formation < 0.4 | 耗竭（减速+垄断） |
| cooling | Δspeed < -0.1 | 正在冷却 |
| flowing | speed > 0.3 | 正在流动 |
| surging | speed > 0.6 | 高能涌动 |

## 7. LLM 辅助信号

主持人 LLM 在输出 `AgendaDecision` 时，额外提供两个信号：

- **emotional_temperature** (0-1)：情绪温度，0=冷静理性，1=高度情绪化
- **perceived_tension** (low/moderate/high/conflict)：感知到的对话张力

这两个信号用于校准规则引擎的判断。规则引擎擅长统计（计数、比率），LLM 擅长理解语义（情绪、意图）。两者互补。

## 8. 调度器三层防线

在信号感知系统之上，调度器引入三层防线机制，确保讨论质量和节奏。

### 8.1 架构

```
┌──────────────┐   ┌──────────────┐   ┌──────────────────┐
│ 第1层 Pre-LLM │ → │ 第2层 LLM    │ → │ 第3层 Post-LLM   │
│ 递增注入     │   │ 有温度的     │   │ 仅拦截明显错误   │
│ 改善信息质量 │   │ 灵活判断     │   │ 最小化干预       │
└──────────────┘   └──────────────┘   └──────────────────┘

基础设施：SchedulingState（轻量状态追踪器）
├── novelty_scores[]          # 每轮新颖度历史
├── consecutive_high_repetition  # 连续高重复轮次（用于日志）
├── speeches_since_novel      # 自上次新颖轮以来的累计发言次数（核心判断依据）
├── speak_history[]           # 近 N 轮发言统计
├── consecutive_silence{}     # 连续未发言轮次
└── rounds_since_anchor       # 距上次回锚轮次
```

### 8.2 CONVERGENCE 收敛递增

用 **novelty_score**（新颖度）衡量收敛质量，按**发言次数**（而非轮次）计数：

```
novelty_score = concept_turnover × 0.5 + kl_normalized × 0.5
```

- `novelty_score > 0.2`：有新东西，不算重复，`speeches_since_novel` 归零
- `novelty_score ≤ 0.2`：高重复，本轮发言次数累加到 `speeches_since_novel`

| 连续低新颖度发言次数 | 第1层注入 | 第3层硬规则 |
|-----------|----------|-----------|
| 6-11 次 | 温和提示（仅供参考，主持人自行判断） | 无 |
| 12-15 次 | 建议转向新子命题 | 无 |
| ≥ 16 次 | 强制 notice 要求话题转移 | 无 |
| ≥ 12 次 | — | 若 phase=CONVERGENCE → 强制 DEEPENING |
| ≥ 20 次 | — | 穷尽提醒 notice（建议评估是否该收敛） |

> 关键：高重复触发的是**话题转移**（回到 DEEPENING），不是 CLOSING。CLOSING 只由时间驱动。
> 注入系统只提供参考，主持人必须根据讨论实际内容独立判断。

### 8.3 CLOSING 时间驱动

closing_window 使用 sqrt 曲线，上界 5 轮：

```
closing_window = max(2, min(5, ceil(sqrt(max_rounds))))
```

| max_rounds | closing_window |
|-----------|---------------|
| 10 | 2 |
| 30 | 3 |
| 50 | 3 |
| 80 | 5 |

当 `rounds_left ≤ closing_window` 时注入尾声提醒，当 `rounds_left ≤ closing_window // 2` 时强制 phase 为 CLOSING。

### 8.4 发言权平衡

**第1层**：注入近 3 轮发言统计，让 LLM 自行判断是否需要平衡。

**第3层**：唯一的硬规则 — 连续沉默保护。阈值动态计算，目标是纯随机下触发概率 < 0.5%：

```
threshold = max(4, ceil(log(0.005) / log(1 - max_speakers/participants)))
```

| 参与者 | 每轮选几人 | 阈值 |
|--------|----------|------|
| 3 | 2 | 4 |
| 5 | 3 | 5 |
| 6 | 3 | 6 |
| 8 | 4 | 6 |

### 8.5 原始问题锚定

渐进式提醒，避免变成噪音：

| 未回锚轮次 | 干预方式 |
|-----------|---------|
| ≥ 6 轮 | 温和注入：agenda_note 中提醒 |
| ≥ 10 轮 | 中等注入：主持人 prompt 加罗盘检查指令 |
| ≥ 15 轮 | 强制：notice 中追加罗盘检查内容 |

## 9. 技术选型说明

| 需求 | 选择 | 原因 |
|------|------|------|
| 分词 | 正则 + jieba（可选） | 零依赖，jieba 可提升精度 |
| 语义相似度 | 关键词重叠（非 Embedding） | Embedding 需加载 500MB 模型 |
| 抽象度 | IDF 均值 + 词长启发式 | 内置 IDF 表，无外部依赖 |
| 立场检测 | 正则标记词（非 NLI 模型） | NLI 模型 ~400MB，标记词够用 |
| 趋势检测 | EMA 滑动窗口 | 纯数学，几行代码 |
| 社区发现 | 暂未实现（需要图论库） | 第一层不引入 networkx |

## 11. 扩展指南

**添加新的原始信号：** 在 `sensors.py` 的 `RawSignals` 中添加新字段，在 `compute_raw_signals()` 中添加计算逻辑。

**添加新的控制信号：** 在 `observer.py` 的 `ControlSignals` 中添加新字段，在 `_map_to_control()` 中添加映射规则。

**添加新的注入规则：** 在 `injector.py` 的 `_build_default_rules()` 中添加新的 `InjectionRule`。

**升级为 Embedding 驱动：** 替换 `sensors.py` 中的 `_measure_direction()` 为 Embedding 余弦相似度计算。其他部分不受影响。
